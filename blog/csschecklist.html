My checklist for the CSS code-not-working situation

I've been messing up with HTML, CSS, Liquid, and Bootstrap a lot recently and I summed up this why-my-CSS-is-not-working list for myself (and for anyone who ever reached this page):

  • Typo
  • Wrong keyword/class name
  • Open div
  • Wrong math
  • Wrong file/page
  • Browser support

More detailed explanations about the list:

Typo

This takes up about 80% of the cases. And it usually happens during midnight or around 3:52 pm and I don't know why ðŸ™‚

Wrong keyword/class name

This happens a lot when I work on different projects in the same day. So at Primary, we have our own styleguide in which every class starts with a "pw-" prefix, but the grid system I'm using for my website doesn't have that. So during the past few weeks when I worked on refreshing my portfolio after work, typing in "pw-" in my text editor felt such a natural thing for muscles when my brain's laying back. A piece of advice would be: trying refreshing your mind before switching to another project.

Open div

You know what I mean.

Wrong math

I once thought 2+5+6=12 and I believed the third div just didn't like the two divs before it. My bad!

Wrong file/page

When you have multiple tabs open and you constantly switch between them the chance of editing a wrong file increases. And sometimes you believe the code should be working but after several refreshes the position of the block still remains the same...that could be you refreshing the wrong page. Try not to have too many tabs open at the same time. I really wish I could "lock" a tab like I lock a layer in Figma. That would make my life much easier.

Browser support

This rarely happens for me but when playing with CSS animation you might want to make sure if your browser/targeted browsers support the property. A reference I really love is the MDN web docs by Mozilla. If you go check on each property page, then at the bottom of each page you'll see a Browser compatibility chart that lists which browsers (desktop and mobile) support the property. It's pretty visualized and you can even find more detailed specifications that may need extra effort to find elsewhere.

Final words: Make use of the inspector wisely, take a break when anxiety occurs.

Happy CSS-ing!
